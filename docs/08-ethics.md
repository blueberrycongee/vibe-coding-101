# 08 引用、版权与责任

## AI 生成代码的责任归属

Vibecoding 带来一个重要问题：**AI 生成的代码，谁负责？**

Boris Cherny 的立场很明确：提交者对代码负全责，不管代码是 AI 生成还是手写。这意味着：
- 你 `git commit` 的代码就是你的代码
- “AI 写的”不是推卸责任的理由
- 出了 bug 或安全漏洞，追责的是提交者

## 基本要求

### 引用规范
- 引用他人观点**必须注明来源**（人物名 + 出处链接）
- 不确定或无法验证的内容要**明确标注**（如”未经证实”、”大意如此”）
- 不传播未经授权的私密信息
- 区分**直接引用**（原文）和**意译**（你的理解）

### AI 生成内容的标注
- 大量使用 AI 生成的文档或代码，建议标注”AI 辅助生成”
- 开源项目中，commit message 中标注 AI co-author 是好实践：
  ```
  Co-Authored-By: Claude <noreply@anthropic.com>
  ```

### 版权注意事项
- AI 可能生成与训练数据中的开源代码相似的片段——注意许可证兼容性
- 不要让 AI 复制粘贴有版权的代码（如 Stack Overflow 回答需遵守 CC BY-SA）
- 使用 AI 生成的代码时，仍需确保符合项目许可证

## 推荐做法

### “观点 + 来源 + 你的理解”结构

在本仓库的笔记中，每条观点都应包含：
```
[人物] 认为 [观点摘要]。
来源：[链接]
我的理解/延伸：[你的补充]
```

### 对争议观点的处理
- 附上**不同来源**的不同观点
- 例如：Karpathy 认为 vibe coding 可以”不看代码”，而 Boris 认为生产代码仍需审查——两者都有道理，适用场景不同

### 安全责任
- AI 生成的代码可能引入安全漏洞（注入、XSS、敏感信息泄露）
- **关键安全路径的代码必须经过人工审查**，不能完全依赖 AI
- 参见：[[human-review-essential]]
