# Swyx: 反对 Vibe Coding，主张 AI Engineering 和 Tiny Teams

## 人物简介
Shawn Wang (Swyx) 是 AI 工程师、Latent Space 播客主编、Cognition 成员。他提出了 "AI Engineer" 概念，是 AI 辅助开发领域的活跃思考者。

## 核心立场：明确反对 Vibe Coding

**Swyx 是 vibe coding 最尖锐的批评者之一**，立场与 Karpathy 和 Levels 截然不同。

### "Slop Attractor"（低质量吸引器）
- 称 vibe coding 是 "slop attractor"——它成为借口，让人在 80% 处停下，忽略最难的 20% 收尾工作
- 认为 Karpathy 创造这个词"对行业造成了净损害"（net disservice），因为它让本不该被美化的东西变得很酷
- 来源：RedMonk 访谈 (2025-07) https://redmonk.com/blog/2025/07/23/shawn-swyx-wang-ai-engineer/

### "RIP Vibe Coding" (2025-11)
- 在 Latent Space 播客宣告 vibe coding 时代结束
- 讨论从 vibe coding 到结构化方法的转变
- 来源：https://podcasts.apple.com/us/podcast/rip-vibe-coding-feb-2025-oct-2025/id1680633614?i=1000735095629

### Tiny Teams 替代方案
- 将 AI Engineer World's Fair 中原定的 "Vibe Coding" 类别改为 **"Tiny Teams"**
- 定义：小而高效、ARR（百万美元）超过员工数的团队
- 已被 levelsio、Cognition、Bolt、Gamma 等实现
- "有人会付钱让你做 AI Engineering，但没人会付钱让你 vibe code"

### "Scaling without Slop" (2026-01)
- 在 AIE Code 大会上"向 Slop 宣战"
- 批评 "Canadian Girlfriend AI Coding" 式的表演性生产力
- 来源：https://www.latent.space/p/2026

## 可提炼的技巧
1. 关注**输出质量**而非体验感受
2. 把自己定位为 **AI Engineer** 而非 vibe coder
3. 用 **Tiny Teams** 框架思考组织效率——小团队 + AI agent = 极高产出
4. Vibe coding 鼓励的是草率而非效率——真正重要的是结果
