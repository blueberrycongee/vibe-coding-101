# Karpathy 提出 "Vibe Coding" 概念

## 方法
- 2025 年 2 月 2 日，Andrej Karpathy 在 X 上首次提出 "vibe coding" 概念
- 核心描述："There's a new kind of coding I call 'vibe coding', where you fully give in to the vibes, embrace exponentials, and forget that the code even exists."
- 具体做法：
  - 用 SuperWhisper 语音转录来表达意图，几乎不手敲代码
  - 使用 Cursor Composer + Sonnet 作为主要工具
  - 遇到错误时直接复制粘贴错误信息给 AI，通常就能修复
  - "I just see stuff, say stuff, run stuff, and copy paste stuff"
  - 对生成的代码只做粗略浏览（半看半不看），主要看结果是否符合预期
  - "the code grows beyond my usual comprehension" — 代码已超出他通常的理解范围

## 适用场景
- 个人的周末项目、throwaway projects
- 快速原型和 POC
- 学习新框架或技术时的探索
- "not quite prompt engineering — I don't have the patience for that"

## 边界
- Karpathy 自己也承认这主要适用于低风险的个人项目
- 不适合生产级代码（Boris Cherny 对此有明确补充：生产代码仍需审查）
- "vibe coding" 的原始定义是一个极端实验，实践中大多数人采用更务实的方式
- 代码理解能力仍然重要——即使不逐行审查，也需要大致理解架构
