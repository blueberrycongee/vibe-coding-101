# Fireship 对 Vibe Coding 的评论

## 人物简介
Jeff Delaney (Fireship) 是知名 YouTube 技术内容创作者（4.1M+ 订阅者），以简洁幽默的风格解释编程概念和技术趋势。

## 核心立场：平衡但谨慎

### "Vibe Coding Mind Virus" (2025-03-26)
- 将 vibe coding 定义为 **"mind virus"（思想病毒）**
- 区分了 **coding**（写代码，基于模式的翻译）和 **programming**（更广泛的创造性解决问题）
- LLM 擅长 coding，但缺乏真正 programming 所需的创造力、经验和上下文理解
- 来源：https://odysee.com/@fireship:6/the-vibe-coding-mind-virus-explained%E2%80%A6:7

### Fireship 的三条 Vibe Coding 规则
1. **选择流行技术栈** — React、Express、Tailwind、Redis 等主流框架，LLM 在这些有大量训练数据的领域表现最好；冷门框架 = 更多错误代码
2. **必须掌握版本控制 (Git)** — AI 可能会删除可用代码，没有 Git 就无法恢复
3. **转变思维为产品经理** — 从写代码者变为提供清晰需求和文档的人

### 关键洞见
- 不理解底层逻辑的代码 = 技术债务
- "You're essentially building on top of a black box"
- 用 AI 可以更快入门，但仍需理解基础原理
- 工具选择应根据场景而非热度

## 可提炼的技巧
1. 用主流、文档丰富的框架配合 AI——LLM 对训练数据丰富的技术栈效果最好
2. 始终用 Git——AI 可能删除可用代码
3. 把自己当产品经理而非码农——清晰的需求比精巧的 prompt 更重要
