# Karpathy 提出 "Agentic Engineering" 替代 Vibe Coding

## 背景
- 2026 年 2 月 4 日，Karpathy 在 vibe coding 一周年之际，提出了新术语 "Agentic Engineering"
- 他承认原始 vibe coding 推文只是 "a shower of thoughts throwaway tweet"，但意外命名了一个时代趋势

## 关键引用

> [!quote] Andrej Karpathy, 2026-02-04
> Personally, my current favorite [term is] 'agentic engineering': 'agentic' because the new default is that you are not writing the code directly 99% of the time, you are orchestrating agents who do and acting as oversight — 'engineering' to emphasize that there is an art & science and expertise to it.

> [!quote] Andrej Karpathy, 2026-02-04
> Programming via LLM agents is increasingly becoming a default workflow for professionals, except with more oversight and scrutiny.

## Vibe Coding vs Agentic Engineering

| 维度 | Vibe Coding | Agentic Engineering |
|------|-----------|-------------------|
| 态度 | 放松、实验性、"给予氛围" | 纪律性、专业性 |
| 审查 | 不读 diff，全部接受 | 有监督、有质量门控 |
| 适用 | 周末原型、一次性项目 | 生产环境、专业开发 |
| 核心技能 | 会说话就行 | 编排 agent + 工程判断 |

## 可提炼的技巧
1. **从"写代码"转变为"编排 agent"**——这是新的核心技能
2. 专业开发需要的不是 vibe coding 的随意，而是**有结构的监督**
3. "Agentic Engineering" 强调这是一门有技艺（art）、有科学（science）、有专业门槛（expertise）的学科
4. 这与 Boris Cherny 的观点高度一致：vibecoding 是场景化的，专业开发需要 plan mode + 审查
